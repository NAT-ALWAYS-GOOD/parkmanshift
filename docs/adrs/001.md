# ADR 001: Plateforme Application Client

**Date:** 30 mars 2026

**Statut:** ACCEPTÉ

**Contributeur(s):** Noam DE MASURE, Tom BOURLARD, Anaelle EA

## Contexte

L'application de réservation de place de parking a besoin de contenir ces fonctionnalités : 
- un employé peut effectuer / modifier / annuler une réservation
- un employé peut effectuer le check-in de sa place réservée via un QR code sur la place ou l'application
- un manager peut consulter les statistiques liées à l'occupation des places 
- un secrétaire peut gérer les utilisateurs et les réservations

## Solutions envisagées

1. Application mobile
- Utilisation d'une application mobile pour mettre à disposition l'ensemble des fonctionnalités.

2. Application web
- Utilisation d'une application web pour mettre à disposition l'ensemble des fonctionnalités.

3. Application web et mobile
- Utilisation d'une application web pour la partie gestion des réservations, des employés et les statistiques.
- Utilisation d'une application mobile pour qu'un employé puisse gérer ses réservations et se check-in.

## Décision

Nous avons écarté l'utilisation d'une application mobile uniquement car les besoins d'un dashboard de statistiques et d'une gestion fine des employés et des réservations étaient difficilement présentatble sur un mobile. 
Nous avons également écarté l'utilisation d'une application web et mobile car les seules fonctionnalités mobile pertinentes seraient le check-in et la gestion de réservation d'une place par un employé. De plus, le check-in peut très bien s'effectuer depuis le mobile sur l'application web après un scan de QR code avec l'appareil photo. 

Nous avons donc opté pour l'application web, cette solution nous permettant de mettre à disposition des différents utilisateurs toutes les fonctionnalités souhaitées.

## Conséquences

### Bénéfices

- Une seule application front à développer et à maintenir.
- Pour les utilisateurs, un seul outil à prendre en main, à un seul endroit; pas besoin d'installer d'application sur des téléphones potentiellement persos.
- Pas de couts liés au déploiement de l'application sur les stores.
- Facilement adaptable au SSO de l'entreprise, contrairement à un usage mobile.

### Inconvénients

- Accessibilité réduite si en déplacement.
